# ソフトクリームバランスゲーム 仕様書

## 概要

コーンにソフトクリームを巻き続け、いかに高く積めるかを競うスマートフォン向けゲーム。プレイヤーはノズルを XZ 平面で動かしてクリームの形を制御し、重心が崩れたら終了する。

## ゲームフロー

```
idle ──（レバー操作）──→ running ──（崩壊検知）──→ collapsed
  ↑                                                    │
  └─────────────（リスタートボタン）─────────────────┘
```

- **idle**: コーンとノズルのみ表示。クリームは出ていない。
- **running**: クリームが一定速度で射出され続ける。崩壊条件を満たすと collapsed に遷移。
- **collapsed**: 最終スコア（積み上げ高さ）とリスタートボタンを表示。

## 操作仕様

### レバー

画面右側に固定したレバー UI。下方向に一定距離ドラッグすると idle → running に遷移する。

### ノズル

`running` 中は画面のドラッグでノズルが動く。トラックパッド準拠で、ドラッグの相対移動量を XZ の移動量に変換する（絶対座標化はしない）。Y 軸（高さ）は固定で、移動範囲は原点中心の円内に制限する。

レバー UI 側にのみ pointerEvents を有効にし、Canvas はタッチイベントをパススルーしない設定で操作とレンダリングの衝突を避ける。

## ゲームオブジェクト

### コーン

画面中央の固定位置。XZ 座標は変化しない。積み上げ高さが一定値を超えるとコーンの Y 座標のみ徐々に下がり、埋まっていくコミカルな演出を行う（クリームの点列は移動させない）。

### ノズル

クリーム射出元。Y 座標固定。XZ 平面内をユーザー操作で移動し、原点中心の円内にクランプする。

### ソフトクリーム

ノズルの軌跡（`Vector3` 点列）を `CatmullRomCurve3` で平滑化し、`TubeGeometry` で押し出してレンダリングする。

- チューブ半径は固定（巻いた量に依存しない）
- 高さ（Y）は経過時間に比例した単調増加。XZ 分布による高さ変化は考慮しない
- 高さの上限なし

## バランスシミュレーション

ソフトボディ計算は行わず、重心ベースの簡略モデルで判定する。

- **重心 (CoM)**: 全クリーム点列の XZ 平均
- **オフセット**: CoM とコーン中心 (原点) の XZ 距離
- **崩壊判定**: オフセットが閾値を超えたら collapsed。閾値は積み上げ高さに反比例して縮小（高く積むほど不安定）
- **傾きフィードバック**: オフセットに比例してクリーム全体を視覚的に傾ける（lean アニメーション）

## UI・画面構成

縦画面スマートフォンを基本とする。PC ブラウザでは画面横幅を固定して中央寄せにし、左右余白を白背景で表示する。

- **HUD**: `running` 中は現在の高さをリアルタイム表示
- **GameOver**: `collapsed` で最終スコアとリスタートボタンを表示

## スコア

崩壊時点の積み上げ高さをスコアとする。

## 技術スタック

| 役割           | 採用技術                               |
| -------------- | -------------------------------------- |
| フレームワーク | React                                  |
| ビルドツール   | Vite                                   |
| 言語           | TypeScript                             |
| 3D レンダリング | @react-three/fiber + @react-three/drei |
| パッケージ管理 | npm                                    |

### コンポーネント構成

```
<App>
  <GameProvider>        // useReducer + useRef で状態管理
    <Canvas>            // r3f
      <Lighting />
      <Cone />
      <CreamTube />     // points[] から TubeGeometry を生成
      <Nozzle />
      <CameraRig />
      <GameLoop />      // 毎フレームの tick / 崩壊検知
    </Canvas>
    <HUD>               // HTML overlay
      <Lever />
      <Score />
      <GameOver />
      <NozzleDragSurface />
    </HUD>
  </GameProvider>
</App>
```

state（`phase`, `score`）は React の reducer、毎フレーム更新される値（クリーム点列、ノズル位置、経過時間など）は `useRef` で保持し、再レンダリングを抑制する。

## スコープ外

- コーンの当たり判定
- 積み方による高さへの厳密な影響（積層計算）
- サウンド
- ハイスコアのローカル保存
